Resonance: Event-Driven Control for Software-Defined Networks

About Resonance

Resonance is an OpenFlow controller that permits event-driven network control. Event-driven network control makes networks easier to manage by automating many tasks that are currently performed by manually modifying multiple distributed device configuration files, which are expressed in low-level, vendor-specific CLI commands.

Resonance can support policies that automatically react to a wide range of events, from fluctuations in traffic volumes to changes in the time of day. Resonance allows network operators to specify network-wide policies in terms of a high-level, event-driven policy model, as opposed to configuring individual network devices with low-level commands.

uCap is a data cap usage management tool for home network users, which uses OpenFlow and OpenWRT. uCap uses the Resonance controller for traffic engineering in home networks. For more information on uCap, please visit this site.

PyResonance

PyResonance is Resonance implemented in Pyretic. Originally, Resonance was implemented in the NOX platform (specifically, in C++). However, programming in the NOX platform is a non-trivial process. Even building the simplest controller program requires multiple steps for setting up the correct environment while the platform does not provide any useful pre-built, high level APIs to programmers. In addition, code readability is low in general for NOX controller programs. Hence, NOX sets a high bar for anyone (e.g., operators, researchers) who wants to implement or understand a Resonance controller with complex policies.

In contrast, Pyretic provides a programmer-friendly environment with high-level APIs and a domain-specific language embedded in Python. We have augmented the original Pyretic code base with many useful features to present an attractive and engaging environment for implementing Resonance controllers. Moreover, Pyretic's modular programming model allows programmers to build a complex network policy by composing multiple network policies together (sequential or parallel).

PyResonance opens a great opportunity to express and enforce network policies that were not easy, if not impossible, in a much simpler fashion with significantly less number of lines of code.
