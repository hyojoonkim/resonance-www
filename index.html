Resonance: Event-Driven Control for Software-Defined Networks

About Resonance

Resonance is an SDN control platform that advocates event-driven network control. Resonance makes networks easier to manage by automating many tasks that are currently performed by manually modifying multiple distributed device configuration files in face of various network events, which are expressed in low-level, vendor-specific CLI commands.

Resonance supports policies that automatically react to a broader range of network events, like fluctuation of traffic volume, intrusion detection, user authenticated, or even change in time-of-day. Resonance uses a Finite State Machine (FSM) model to express such operational tasks; states represent different actions or behavior depending on network condition, and transitions between states represent reactions to dynamic network events.

Kinetic

Kinetic is Resonance implemented with Pyretic. Originally, Resonance was implemented in the NOX platform (specifically, in C++). Pyretic provides a better programmer-friendly environment with high-level APIs, libraries, and constructs. Pyretic's composition operators (parallel and sequential) add great value by enabling composition of multiple modular network programs. Kinetic is built on top of Pyretic, presenting an attractive and engaging environment for implementing Resonance-based controllers.

Using Kinetic, programmers can implement and enforce a complex, network-wide policy that can dynamically change network behavior based on various types of network events. Using composition operators, it is also possible to compose multiple finite state machine based network tasks together.

Kinetic supports verification of network policies using the NuSMV symbolic model checker. Kinetic automatically generates a NuSMV input from the program written by the programmer/operator, and verifies logic statements written in CTL (Computation Tree Logic).
